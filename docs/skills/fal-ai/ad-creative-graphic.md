# Graphic Ad Creative Generator

Generate professional Meta/Instagram ad images using **nano-banana** (Gemini text-to-image). ~$0.07/image at 1K, ~$0.10 at 2K.

## When to Use This Skill vs `ad-creative`

| This Skill (`ad-creative-graphic`) | `ad-creative` |
|-------------------------------------|---------------|
| Bold text, CTA buttons, icons | Photorealistic scenes |
| Comparison layouts | Jay composited into photos |
| Feature grids, social proof | Lifestyle/product photography |
| **~$0.07/image via nano-banana** | ~$1.50/campaign via fal.ai |

## Ad Types

### `comparison` — Old Way vs New Way
**Best for:** ToF awareness
**Layout:** Split two-column, left side negative (red X marks), right side positive (green checks)
**When:** Positioning against competitors or the old manual way

### `feature-grid` — Multi-Feature Showcase
**Best for:** MoF consideration
**Layout:** Bold headline + grid of icon+label cards + offer bar + CTA
**When:** Showcasing multiple features or tools in one ad

### `social-proof` — Testimonial + Metrics
**Best for:** BoF conversion
**Layout:** Quote + inline metrics (dashboard-style, NOT equal cards) + CTA
**When:** Leveraging customer results and testimonials

### `minimal-text` — Bold Statement
**Best for:** ToF awareness, retargeting
**Layout:** Giant headline filling entire frame + accent words + CTA
**When:** A single powerful statement needs to dominate the feed

### `founder-proof` — Founder Photo + Proof Screenshot
**Best for:** All funnel stages — builds trust through personal brand + evidence
**Layout:** Hot pink headline banner (1/3 height) + Flux Lora photo of Jay (left ~40%) + large proof screenshot (right ~60%) + CTA
**When:** You want to combine personal brand credibility with real proof (dashboard, calendar, Slack DM). Requires a Flux Lora photo of Jay as one `-r` reference.
**Input:** Use `-r jay-photo.png` for the founder photo, optionally `-r dashboard.png` for a real dashboard screenshot
**Proof types allowed:**
- Well-known UIs generated by Gemini: Google Calendar, iMessage, Slack, Google Sheets, WhatsApp
- Real dashboard screenshots passed as `-r` reference (e.g., Instantly, CRM dashboards)
- **NEVER use payment dashboards** (Stripe, PayPal, etc.) — not allowed on Meta
- **NEVER prompt Gemini to generate company-specific dashboards** (Instantly, Mailchimp, HubSpot) — only pass real screenshots as `-r` references

## Workflow

### Step 1: Gather Context
Collect offer, funnel stage, key message, and format from user.

### Step 2: Select Ad Type
Use the funnel stage guide above. Suggest 1-2 options with reasoning.

### Step 3: Write Ad Copy
- Headline: 15 words max, uppercase, identify 1-3 accent words for hot pink
- Subtext: 1-2 sentences, conversational
- CTA: Action verb + benefit ("GET STARTED FREE", "SEE HOW IT WORKS")
- Check against CLAUDE.md banned words list

### Step 4: Construct Nano-Banana Prompt
Use the appropriate **Prompt Template** below. Fill in placeholders from the brief.

### Step 5: Generate Image
```bash
nano-banana "PROMPT" -a 4:5 -s 1K -d output/ads/graphic/ -o <descriptive-name>
```

**CLI flags:**
- `-a 4:5` — aspect ratio (4:5 portrait default, 1:1 square, 9:16 story)
- `-s 1K` — size (1K default, 2K only when user explicitly asks for production-grade)
- `-d output/ads/graphic/` — output directory
- `-o name` — output filename (no extension)
- `-r reference.png` — reference image for style remixing

### Step 6: Review & Refine
Open output in Preview.app (`open -a Preview output/ads/graphic/<name>.png`). Check against Design Quality Rules below. If issues found, edit the prompt and regenerate.

## Prompt Engineering Rules

### Always Include
- "Professional Meta ad creative for social media advertising" — sets commercial context
- "Clean white background (#FFFFFF)" — white bg default for ads
- "hot pink color #ED0D51" — locks brand accent
- "Big Shoulders Black" for headlines, "Manrope" for body — brand fonts per style guide
- "Clean modern editorial typography" — prevents decorative AI patterns
- Explicit negative constraints: "no gradients, no stock photos"
- Enforce correct spelling — for words that Gemini commonly garbles, repeat the word twice in the prompt (e.g., "the word EMAILS — spelled E.M.A.I.L.S") and add a negative constraint ("Do NOT misspell EMAILS"). Never use dashes between letters in the visible ad text — dashes will render literally.
- Specify headline line breaks explicitly for multi-phrase headlines (e.g., "Line 1: COLD EMAILS OUT. Line 2: CALLS IN.")

### Headline Instructions
- Always specify "large bold white uppercase text, taking up at least one-third of the image height"
- Name specific accent words and their color
- The headline instruction must come early in the prompt (Gemini prioritizes early tokens)

### Testimonials & Stats Must Look Screenshotted
- Testimonials must appear as embedded screenshots of real messaging apps (Slack DM, iMessage, email) — with chat bubbles, timestamps, platform UI chrome
- Stats/metrics must appear as embedded screenshots of real dashboards (CRM, Google Analytics) — with table rows, column headers, chart labels
- Screenshots should look slightly rotated or drop-shadowed, as if pasted onto the ad
- NEVER style testimonials as designed quote cards or metrics as equal-width colored boxes
- **NEVER use payment dashboards** (Stripe, PayPal) — banned on Meta ads
- **NEVER prompt Gemini to generate company-specific dashboards** (Instantly, HubSpot, Mailchimp) — only use well-known UIs (Google Calendar, Slack, iMessage, Google Sheets, WhatsApp) or pass real screenshots as `-r` references
- Proof screenshots must be large enough to read data at mobile phone size — zoom in on key metrics

### Anti-AI-Slop in Prompts
- "NOT as 3 identical equal-width colored cards" — prevents the #1 AI fingerprint
- "no decorative filler" — prevents empty visual elements
- "varied card sizes preferred over identical boxes" — breaks symmetry
- "screenshot of a real analytics dashboard" — makes metrics look authentic
- "screenshot of a real Slack DM or iMessage" — makes testimonials look authentic

### Reference Image Strategy
Use `-r reference.png` when:
- Remixing a proven competitor ad pattern
- Matching a specific layout/composition style
- Iterating on a previous generation that was close

## Visual Artifact Rules

### The "Screenshotted Artifact" Principle
Any content element that would naturally exist as a real document, note, or display must be prompted as a **screenshotted real artifact** — not styled text on the ad background.

| Content Type | Prompt As | NOT As |
|-------------|-----------|--------|
| Checklists / bullet lists | Notepad, whiteboard, sticky note | Styled text with emoji checkmarks |
| Math / equations / stats | Whiteboard, chalkboard, spreadsheet cell | Bold text with arrows |
| Reports / diagnoses | Clipboard, medical form, printed document | Text in a gray box |
| Recipes / ingredients | Handwritten recipe card, cookbook page | Bulleted list in a bordered box |
| Testimonials / quotes | Slack DM, iMessage, tweet screenshot | Quote card with quotation marks |
| Dashboards / metrics | REAL screenshot passed as `-r` reference | Gemini-generated fake cards |
| Spreadsheets | Specify "Google Sheets" or "Microsoft Excel" | Generic "spreadsheet" |
| Bold statements | Tweet mockup, notepad page, handwritten note | Giant text on plain white |

### When to Use `-r` Reference Images (MANDATORY)
- **Dashboards**: Always pass a real screenshot from `public/ads/assets/lead gen/demo/` — never let Gemini generate dashboards
- **Spreadsheets**: For the "after" view, use a real Instantly or CRM dashboard as `-r`

## Design Quality Rules

These rules apply to ALL ad output and are used for vision grading.

### Headline Dominance
- Headline + subheadline must occupy at least 1/3 of the total image height
- Headlines should feel like they OWN the frame — never small or squeezed
- If the headline doesn't dominate, add more emphasis in the prompt

### Stats & Metrics
- Stats must look like a real spreadsheet or dashboard — NOT equal-width colored cards
- Avoid 3 identical metric boxes in a row (screams AI-generated)
- Better patterns: inline text with bold numbers, dashboard-style rows, asymmetric blocks
- Numbers should feel pulled from a real report, not designed as decorative elements

### Anti-AI-Slop Rules

**Color:**
- NO purple/blue gradients — brand colors only (`#ED0D51`, `#FFFFFF`, `#000000`)
- 60-30-10 ratio: 60% white bg, 30% black/dark gray text, 10% `#ED0D51` accent
- Accent on: 1-3 headline words + CTA pill — nothing else

**Layout:**
- NO 3-column equal grids with identical cards (biggest AI fingerprint)
- Asymmetric layouts preferred over symmetric grids
- One accent element per section max
- Vary card sizes — not all identical heights

**Content:**
- Real data always: "$295/mo", "47 calls", "14 days" — never "XX" or round placeholders
- No blob people, Corporate Memphis, stock placeholders, or Lorem Ipsum
- Specific names and copy, not generic
- If the entire ad is just styled text on white with no visual artifact (screenshot, notepad, device, photo), it's too generic — wrap content in a real-world artifact per the Visual Artifact Rules

**Visual:**
- Every element serves a purpose — no decorative filler
- No warped, blurry, or AI-distorted text/letterforms
- All text must be sharp and legible at mobile size
- No large empty areas — content fills the frame with intentional spacing

### Section Purpose Rule
- Every section in the ad must serve a clear, obvious purpose
- NO filler sections: fake bar charts, abstract "proof" areas, placeholder screenshots
- If a section can't be filled with real content, remove it and give that space to the headline

### Pre-Delivery Checklist
- [ ] White background, hot pink accent in headline AND CTA
- [ ] Headline dominates — at least 1/3 of frame height
- [ ] Headlines use Big Shoulders Black (bold condensed), body uses Manrope
- [ ] Testimonials look like real screenshots (Slack/iMessage/email chrome visible)
- [ ] Stats look like real dashboard screenshots (table rows, chart labels visible) — NO payment dashboards
- [ ] Brand bar present with "Lead Gen Jay" + "@leadgenjay"
- [ ] No AI-slop patterns, no banned words
- [ ] Max 15 words per section
- [ ] Text is sharp and legible — no warped letterforms
- [ ] No dead space — content fills frame
- [ ] Open in Preview.app and visually confirm quality

## Variations

For A/B testing, create 2-3 variations per ad:
1. **Copy variation:** Same ad type, different headline/CTA
2. **Format variation:** Same copy, different aspect ratio (4:5 + 1:1)
3. **Model variation:** Same prompt, flash vs pro model (`-m pro`)

Generate all variations in sequence:
```bash
nano-banana "PROMPT_V1" -a 4:5 -s 1K -d output/ads/graphic/ -o ad-v1
nano-banana "PROMPT_V2" -a 4:5 -s 1K -d output/ads/graphic/ -o ad-v2
nano-banana "PROMPT_V1" -a 1:1 -s 1K -d output/ads/graphic/ -o ad-v1-square
```
